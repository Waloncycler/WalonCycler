Hexo常用命令

简写指令：

简写指令 全称指令
hexo n “我的博文” hexo new（post） “我的博文”
hexo p hexo publish
hexo g hexo generate
hexo s hexo server
hexo d hexo deploy
hexo deploy -g hexo deploy –generate
hexo generate -d hexo generate –deploy
hexo clean hexo clean
git –version git –version

**注：git –version , hexo clean 没有简写 **

指令说明：

指令 功能
hexo -v 查看hexo版本号
git –version 查看git版本号
npm -v 查看npm版本号
node -v 查看node.js版本号
nmp install hexo -g 安装
npm update hexo -g 升级
hexo init “文件夹名X” 初始化X文件夹名称
hexo d 将本地数据部署到远端服务器
hexo g 生成静态网页
hexo clean 清除缓存
hexo server Hexo 会监视文件变动并自动更新
hexo server -s 以静态模式启动
hexo server -p 8080 更改访问端口（默认4000）
hexo server -i IP地址 自定义 IP

注：执行$ hexo g后会在站点根目录下生成public文件夹, hexo会将”/blog/source/“ 下面的.md后缀的文件编译为.html后缀的文件,存放在”/blog/public/ “ 路径下)

  • hexo publish [layout] <title> #通过 publish 命令将草稿移动到 source/_posts 文件夹,如:$ hexo publish [layout] <title>,草稿默认是不会显示在页面中的，可在执行时加上 --draft 参数，或是把 render_drafts 参数设为 true来预览草稿。

  • hexo new aaa "bbb" # 新建一篇文章,文章名称和标题分别为bbb.md 和 bbb. 文章采用aaa布局, 此时会在站点根目录下的—->source—–>_post文件夹下生成bbb.md文件, bbb.md文件的顶部(—–分割线上方区域,也称作Front matter区),生成
